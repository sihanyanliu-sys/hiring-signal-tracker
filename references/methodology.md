# 方法论:招聘信号如何服务于价值投资研究

本文是 hiring-signal-tracker 的分析方法论。任何 agent 或分析师在解读报告数字前应先读完本文。

## 1. 逻辑勾稽链

企业内在价值 = 未来自由现金流折现,驱动变量是营收增长率、利润率、再投资回报率。
招聘数据挂钩的链条:

1. **招聘是管理层"真金白银的表态"**。电话会上说"AI 是战略重点"是免费的;批准
   200 个 headcount 是每年数千万美元的固定支出承诺,只有内部看到需求可见性
   (pipeline、客户意向)时才会发生。招聘数据的本质是把管理层的内部信息提前
   泄露给外部观察者,领先财报确认 1~2 个季度。
2. **不同岗位对应价值链不同环节,信号强度不同**(见第 3 节)。
3. **信号必须与硬数据勾稽**(见第 5 节),单独使用会误判。

学术背书:Babina, Fedyk, He, Hodson 等人用招聘 JD 技能要求测量企业 AI 投入,
发现该指标预测后续营收增长与市场份额扩张。技能维度优于岗位头衔维度:头衔可以
蹭热点改名,JD 里的技能栈意味着面试真考、团队真做。

## 2. 指标口径与设计理由

| 指标 | 口径 | 为什么这样设计 |
|---|---|---|
| 加权技能密度 | 每职位命中 AI 技能词的加权和均值;模型/应用类 ×3、数据/通用类 ×2、口号词 ×1;词条按职位去重 | 具体技术栈是有成本的承诺,口号是免费的;去重防止同词刷屏 |
| AI 参与率 | 命中实质类别(非口号)的职位占比 | 剔除"passion for AI"式废话 |
| AI 方向(模型/应用) | AI 参与职位中命中模型自建类 vs 应用集成类的占比 | 自建模型(重资本、高风险高壁垒)与调 API 做应用(轻资本、快变现)是两种战略,估值含义不同 |
| 需求验证型岗位占比 | FDE / Solutions / 销售工程等岗位占比 | 这类岗位按已签或将签合同配置,是"需求已被验证"的信号,直接指向 2~4 个季度内营收 |
| 非技术岗 AI 渗透率 | 销售/市场/财务等岗位 JD 含实质 AI 技能要求的比例 | AI 从实验室进入业务基础设施的标志,利润率影响由此开始;仅对有一定非技术岗规模的公司有意义 |
| 高管+总监占比 | VP/Chief/Head of/Director 头衔职位占比 | 招高管 = 开新业务线的先导;大量招初级 = 进入规模化执行期 |
| AI 薪酬溢价 | AI 岗位薪酬中位数 / 非 AI 岗位 - 1(来自 JD 公示薪酬带,加州/纽约州等强制披露) | 投入强度最"货币化"的测量;样本 <3 时不计算 |
| 供应商信号 | JD 中出现的第三方 AI 供应商(Anthropic/OpenAI/Bedrock 等) | 反映"在给谁交钱",汇总可读出 AI 供应链站队图 |
| 消失职位 / 红旗 | 与上次快照按 URL 差分;某部门下架 ≥5 且 ≥30% 触发红旗 | 反向信号比正向可靠:公司有动机夸大扩张,没有动机假装收缩 |
| 新地点 / 新部门 | location/department 相对历史全集的首次出现 | 领先于管理层宣布进入新市场/新业务线 |
| 模板词抑制 | 出现在公司 ≥90% JD 的词条自动剔除(职位数 <20 不启用) | 实测 Braze 把 "BrazeAI™... AI agents" 塞进全部 227 份 JD——那是市场部文案,不是技能需求 |

## 3. 信号质量分层

- **最强**:需求验证型岗位变化(合同驱动)、红旗(收缩藏不住)。
- **较强**:非技术岗 AI 渗透变化、AI 薪酬溢价变化、新地点/新部门首现。
- **中等**:技能密度与 AI 参与率变化(方向可信,幅度受 JD 风格影响)。
- **最弱,几乎不给权重**:研究型岗位数、任何跨公司的绝对值比较。

**铁律:同公司时间序列 > 同主题内横比 > 跨主题横比(基本无效)。**
Palantir 密度低是 Lever JD 写作风格短小所致(测量偏差),不代表 AI 投入低。

## 4. 已知局限与坑

- 词典是英文的,只适用于英文 JD;A 股/港股需另建词典。
- Workday/自建系统的公司(Snowflake、CrowdStrike、Atlassian 等)采不到。
- to C 公司(如 Duolingo)的需求验证型占比天然极低,该指标只对 B2B 有意义。
- 词条误匹配残留:如 "Gemini" 在加密行业可能指交易所(已收紧正则,仍需人工复核
  vendor 列的反常值)。
- 招聘 ≠ 盈利:烧钱扩张与盈利扩张发同样的 JD,必须结合现金流与单位经济看。
- 另类数据卖方(Revelio、LinkUp)服务对冲基金多年,纯公开信号的独家性有限;
  本工具的价值在证据链的时间优势与言行校验,不在"独家 alpha"。
- 修改词典后,密度类指标与历史不可比,应归档 history.csv 重建基线。

## 5. 战略动向雷达:跳跃检测阈值

报告顶部的"战略动向雷达"对比上次快照,超过以下阈值即报告(按严重度加权排序)。
阈值是经验起点,可在 scripts/pipeline.py 的 JUMP_RULES 中调整:

| 信号 | 阈值 | 严重度 | 战略含义 |
|---|---|---|---|
| 收缩预警 | 在招职位 ≤ -10% | 3 | 需求走弱或成本收缩,优先核查 |
| 红旗 | 某部门下架 ≥5 且 ≥30% | 3 | 业务线收缩,先于公告 |
| 需求验证跃升 | 需求验证型占比 +5pp | 3 | 合同驱动扩张,指向 2~4 季度营收 |
| 扩张加速 | 在招职位 ≥ +10% | 2 | 整体扩张,需分辨烧钱 vs 盈利扩张 |
| AI 投入跃升/降温 | 技能密度 ±25%(基数 ≥0.5) | 2 | AI 战略提速/降级 |
| 转向模型自建 | 模型自建占比 +10pp | 2 | 转重资本策略,勾稽资本开支 |
| 高管招聘潮 | 高管+总监占比 +3pp | 2 | 新业务线先导 |
| 新市场扩张 | 新地点 ≥3 | 2 | 地理扩张,先于管理层宣布 |
| 转向应用集成 | 模型自建占比 -10pp | 1 | 转轻资本策略 |
| AI 渗透深化 | 非技术岗渗透 +8pp | 1 | AI 进入业务基础设施 |

周度数据有噪声:单周跳跃是线索,连续 2~3 期同向变化才是趋势;所有结论仍需
过第 6 节的勾稽工作流。

## 6. 勾稽工作流(每次报告后的人工/agent 步骤)

1. 挑出信号显著变化的公司(密度、需求验证型、红旗)。
2. 对照最近 10-Q/10-K:营收增速、RPO/递延收入、资本开支方向是否印证。
   - 需求验证型 ↑ + RPO 加速 = 扩张证据成立;RPO 停滞 = 可能是救火招聘。
3. 对照最近电话会:管理层 AI 表述的具体度(报客户数/收入 vs 谈愿景)与招聘
   动作是否一致。言行不一本身就是研究发现(多为负面)。
4. 结论只到"修正基本面假设"为止:调整增长/利润率假设 → 重估内在价值 → 与
   市价比较。招聘信号永远不直接生成买卖决策。

## 7. 信号有效性验证(项目的"生死检验")

积累 ≥2 个季度数据后,回答可证伪的问题:**"需求验证型岗位/技能密度显著增长的
公司,后续 2~4 个季度营收是否系统性超出分析师共识?"** 按季度滚动记录信号与
实际财报差异;若两个季度后信号对营收超预期无预测力,应停止投入并如实告知用户。
Wayback Machine 上的历史招聘页快照可部分重建过去序列辅助回验。
